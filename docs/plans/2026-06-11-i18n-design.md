# Design: Full i18n (Norwegian + English)

**Date:** 2026-06-11
**Status:** Approved

## Goal

Translate the entire public-facing recipes app into Norwegian (Bokmål) and English —
both UI chrome and recipe content — using Astro's built-in i18n for routing. Content
lives in JSON files and git; no database is involved in the content or delivery path.
The static S3 + CloudFront deploy stays unchanged.

## Decisions

| Question | Decision |
|---|---|
| What gets translated | All text in the public app: UI chrome **and** recipe content |
| Content source | JSON files + git (no DB for content) |
| Translation production | Translated directly into static JSON (in-conversation), then hand-edit; review via git diff. No script. |
| Default locale code | `nb` (Norwegian Bokmål) |
| URL layout | Symmetric: `/nb/…` and `/en/…`; `/` redirects to `/nb/` |
| Scope | Public pages only (index + recipe detail). Login/admin excluded. |
| Deploy | Unchanged: static build → `s3 sync dist/client` → CloudFront |

## 1. Locales & routing

Astro built-in i18n config in `astro.config.mjs`:

```js
i18n: {
  locales: ['nb', 'en'],
  defaultLocale: 'nb',
  routing: { prefixDefaultLocale: true },
}
```

- Symmetric prefixes: Norwegian at `/nb/`, `/nb/recipe/:id`; English at `/en/`, `/en/recipe/:id`.
- Page files live under `src/pages/nb/` and `src/pages/en/` (thin route files).
- The root `/` is a static meta-refresh redirect page (`src/pages/index.astro`) pointing to
  `/nb/`. A config-level redirect is **not** used because the node adapter handles
  `redirects` at runtime and emits nothing into the static `dist/client`.
- `Layout.astro` `<html lang>` becomes dynamic from `Astro.currentLocale` (and `lang="no"` → `lang="nb"`).
- Header gains a language switcher (NB | EN) linking to the current page's counterpart
  via Astro's `getRelativeLocaleUrl`.

## 2. UI text (chrome)

- Dictionaries: `src/i18n/nb.json` + `src/i18n/en.json`.
- Helper `t(locale, key)` in `src/i18n/index.ts`.
- Strings to extract (from `index.astro`, `recipe/[id].astro`, `Layout.astro`):
  search placeholder, `"{n} oppskrifter"` / `"{n} av {total} oppskrifter"`,
  `"Ingen oppskrifter funnet."`, header title `"Oppskrifter"`, icon `title`s
  (Kalorier / Protein / Tilberedningstid), units (`kcal`, `g protein`, `min`).
- Login/admin pages are **out of scope** — not part of the static deploy.

## 3. Recipe content model

Numbers (kcal, protein, cook time) are language-independent and must not be duplicated.

- `src/data/recipes.nb.json` — **source of truth**, full recipes (all fields).
  (Renamed from the current `recipes.json`.)
- `src/data/recipes.en.json` — **only translatable text** per recipe `id`:
  `title`, `description`, `category`, `tags`, `ingredients`, `steps`.
- The build merges nb's numeric/structural fields with en's text by `id`.
  English never stores numbers, so they cannot drift.
- **Tag/category consistency:** `src/data/glossary.json` maps each nb tag/category to one
  canonical en label, so a given tag translates identically everywhere and the `/en/`
  filters group correctly.

## 4. Page structure

- Mirror routes under `src/pages/nb/` and `src/pages/en/`, plus the root redirect at
  `src/pages/index.astro`.
- Route files stay **thin** and delegate to shared components (`RecipeList`,
  `RecipeDetail`); only the locale differs, so no markup duplication.
- `getStaticPaths` reads the locale-merged data per locale.

## 5. Translation production

- No script. With only 42 recipes, the English text (`recipes.en.json`) and the tag/category
  `glossary.json` are translated directly into static files and committed.
- Review flow = the git diff on `recipes.en.json` / `glossary.json`. No DB, no API key, no
  build-time dependency on the Anthropic SDK.
- **Future recipes:** translate the new entries the same way (ask Claude, or hand-translate).
  A repeatable `pnpm run translate` script is deliberately deferred until volume justifies it.

## 6. Deploy

- **Unchanged.** Still `pnpm build` → `aws s3 sync dist/client/` → CloudFront invalidation.
- The existing CloudFront rewrite function already maps `/en/…` → `/en/…/index.html`,
  so no Terraform change and no new CI secrets.

## 7. Notes / out of scope

- DB / auth / admin are left untouched but are now redundant for the public site.
  Retiring them is a separate decision, not part of this work.
- Testing: unit tests for the `t()` helper and the build-time nb/en merge (vitest is
  already set up).
